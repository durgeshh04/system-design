# ISP:

- ISP stands for Interface Seggregation Principle.
- "No client should be forced to depend on interfaces they do not use."

## Advantages:

1. Improved Maintainability:

- Promotes smaller, easier to understand interfaces

2. Reduced Impact of Changes:

- Changes in one interface don't affect unrelated classes

3. Increased Flexibility and Reusability

- Specific roles can be combined in different ways

4. Stronger Encapsulation

- Limits the exposure of classes to just needed methods

5. Better Testing

- Easier to mock for unit testing

6. Better Compliance with Other Principles

- Leads to better compliance with other design principles
